Iniciar evento

Un evento de inicio indica el lugar donde y posiblemente por qué comienza un proceso. Dado que el evento de inicio se utiliza para iniciar un proceso, no tiene ningún flujo de secuencia entrante.

 

Ninguno

Tipos de puerta de enlace

En el modelado de procesos de negocio con BPMN, existen varios tipos de puerta de enlace para diferentes tipos de comportamiento de control, como tomar decisiones, bifurcar, fusionar, bifurcar y unir.

 

Basado en eventos
Se puede utilizar una puerta de enlace basada en eventos para modelar las rutas alternativas que siguen la puerta de enlace y que se basan en eventos que ocurren en lugar de la expresión del flujo

 

 Recibe la Pizza : Intermediate Event

 

Evento intermedio

Un evento intermedio indica dónde sucede algo entre el evento de inicio y el final de un proceso. Puede usar un evento intermedio para mostrar dónde se reciben o envían los mensajes, mostrar el retraso necesario, realizar el manejo de excepciones y mostrar la necesidad de compensación. Puede colocar un intermedio incluso en dos lugares: adjuntando el límite de la tarea / subproceso, flujo normal (es decir, conectado desde un flujo sin adjuntar a una actividad).

 60 Minutos : Intermediate Event

 

Evento intermedio

Un evento intermedio indica dónde sucede algo entre el evento de inicio y el final de un proceso. Puede usar un evento intermedio para mostrar dónde se reciben o envían los mensajes, mostrar el retraso necesario, realizar el manejo de excepciones y mostrar la necesidad de compensación. Puede colocar un intermedio incluso en dos lugares: adjuntando el límite de la tarea / subproceso, flujo normal (es decir, conectado desde un flujo sin adjuntar a una actividad).

 

Temporizador
Este disparador actúa como un mecanismo de retraso en una fecha-hora o ciclo específico (por ejemplo, todos los viernes). Solo puede usar un evento intermedio de temporizador en un flujo normal.

 Hambre Satisfecha : End Event

 

Evento final

A diferencia del evento de inicio, el evento de finalización indica dónde terminará un proceso. Dado que el evento final se utiliza para terminar un proceso, no tiene ningún flujo de secuencia saliente.

 

Ninguno
El evento none end no tiene un resultado definido.
